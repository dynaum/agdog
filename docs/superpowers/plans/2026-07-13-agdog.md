# agdog Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `agdog`, a Rust/ratatui terminal monitor that groups CPU/RAM/GPU/VRAM by the agent that owns each process and flags stuck/runaway jobs.

**Architecture:** A tick loop pulls `ResourceSample`s from pluggable `Collector` backends (system via `sysinfo`; GPU via a trait with mock, NVML, and macOS implementations). An attribution engine maps processes to agents, a classifier assigns semantic state, and a ratatui renderer draws the grouped view. A Unix-socket server streams state-change events as JSON.

**Tech Stack:** Rust 2024, ratatui + crossterm (TUI), sysinfo (system metrics), nvml-wrapper (NVIDIA, feature-gated), serde/serde_json (socket protocol), clap (CLI).

## Global Constraints

- Rust edition 2024, crate name `agdog`, binary `agdog`.
- Single static binary. No network calls. No business logic in build scripts.
- Every backend is behind a trait and MUST fall back to the mock backend when unavailable, so the binary always runs on any machine (including an Apple Silicon Mac with no NVIDIA GPU).
- GPU backends are `#[cfg(feature = ...)]` gated: `nvml` (NVIDIA), `macos` (Apple Silicon). Default build uses the mock backend and always compiles.
- TDD: write the failing test first for every deterministic unit (data model, attribution, classifier, socket protocol). Platform backends that need real hardware are validated by a smoke run plus a mock-parity test, not by asserting live GPU values.
- Commit after every task. Keep files focused: one responsibility per module.
- Before each commit run `cargo fmt` and ensure `cargo build` and `cargo test` pass. `cargo clippy` is advisory during development (a crate-level `#![allow(dead_code)]` covers not-yet-consumed items). Enforce `clippy -- -D warnings` only in Task 14 CI, after removing that allow.
- **Loop bookkeeping:** a task is done when its work is committed AND pushed. Mark progress by appending ` âś…` to the task's `### Task N:` header. The next iteration implements the first task header without a âś…. If a step genuinely cannot be completed in this environment (e.g. no NVIDIA hardware for Task 11), ensure it COMPILES behind its feature flag, note the deferral in the commit message, mark the header âś…, and continue â€” never stall the loop or leave `master` non-building.

## File Structure

```
src/
  main.rs            # CLI entry (clap): default TUI, `agdog watch` subcommand
  app.rs             # App state + tick loop, holds Vec<Agent> and history
  model.rs           # Agent, AgentState, AgentKind, ResourceSample, AgentView, Event
  collect/
    mod.rs           # Collector trait + CollectorSet, ResourceSample assembly
    system.rs        # sysinfo-based process/CPU/RAM collector
    gpu.rs           # GpuCollector trait + GpuSample
    gpu_mock.rs      # deterministic fake GPU backend (default)
    gpu_nvml.rs      # NVIDIA backend (#[cfg(feature = "nvml")])
    gpu_macos.rs     # Apple Silicon backend (#[cfg(feature = "macos")])
  attribute.rs       # process -> agent attribution engine
  classify.rs        # semantic state classifier
  socket.rs          # Unix-socket event server + JSON protocol
  ui/
    mod.rs           # top-level render(frame, &App)
    summary.rs       # aggregate strip
    gpus.rs          # per-GPU panels
    agents.rs        # grouped agents table
    detail.rs        # selected-agent detail pane
    alerts.rs        # alerts pane
    footer.rs        # keybind footer
tests/
  attribution.rs
  classify.rs
  socket.rs
```
